# 深度学习中的精度理解

## 1. 什么是精度（Precision）？

精度指的是数值在计算机中的表示方式，决定了数字的存储格式和计算方式。

### 1.1 常见精度类型

| 精度类型 | 位数 | 符号位 | 指数位 | 尾数位 | 数值范围 | 精度 |
|---------|------|--------|--------|--------|----------|------|
| FP32 (Float32) | 32 | 1 | 8 | 23 | ±3.4×10³⁸ | ~7位十进制 |
| FP16 (Float16) | 16 | 1 | 5 | 10 | ±65504 | ~3位十进制 |
| BF16 (BFloat16) | 16 | 1 | 8 | 7 | ±3.4×10³⁸ | ~2位十进制 |
| FP8 (E4M3) | 8 | 1 | 4 | 3 | ±448 | ~1位十进制 |
| FP8 (E5M2) | 8 | 1 | 5 | 2 | ±57344 | ~1位十进制 |
| INT8 | 8 | 1 | - | 7 | -128~127 | 整数 |
| INT4 | 4 | 1 | - | 3 | -8~7 | 整数 |

### 1.2 精度的组成部分

以 FP32 为例：
```
符号位(1bit) | 指数位(8bits) | 尾数位(23bits)
    S       |      E        |       M
```

- **符号位**：表示正负
- **指数位**：决定数值范围（能表示多大/多小的数）
- **尾数位**：决定精度（小数点后能表示多少位）

### 1.3 指数位详解

**指数位是什么？**

指数位（Exponent）是浮点数表示中用来决定**数值范围**的部分，它表示数字的"量级"或"规模"。

**核心概念：**

浮点数使用**科学计数法**来表示数字：

```
数值 = 符号 × 尾数 × 2^指数

例如：
3.14159 = 1 × 1.5708 × 2^1
0.000123 = 1 × 1.0039 × 2^(-15)
65000 = 1 × 1.9849 × 2^15
```

**指数位就是存储这个"2的多少次方"的部分。**

**为什么需要指数位？**

指数位决定了浮点数能表示的**数值范围**：

- **大指数**：可以表示很大的数（如 10³⁸）
- **小指数**：可以表示很小的数（如 10⁻³⁸）
- **指数位越多**：数值范围越大

**指数位的工作原理：**

以 FP32 为例：

```
FP32: [符号位(1)] [指数位(8)] [尾数位(23)]

指数位有 8 位 → 可以表示 2^8 = 256 个不同的值 (0-255)

但实际使用时有偏移量（bias）：
- 存储的指数值 E = 0-255
- 实际指数 = E - 127（偏移量）
- 所以实际指数范围：-127 到 +128
```

**为什么要有偏移量（bias）？**

偏移量让指数可以表示**负数**，而不需要单独的符号位：

```
存储值 E    实际指数 (E-127)    含义
0          -127              最小的数（接近0）
127        0                 1.xxx × 2^0 = 1.xxx
128        1                 1.xxx × 2^1 = 2.xxx
255        128               最大的数
```

**实际例子：**

```
例子1：π = 3.14159265
  FP32 指数位 E = 10000000₂ = 128₁₀
  实际指数 = 128 - 127 = 1
  意思是：π ≈ 1.5708 × 2^1 = 3.1416

例子2：小数 0.000123
  FP32 指数位 E = 01110000₂ = 112₁₀
  实际指数 = 112 - 127 = -15
  意思是：0.000123 ≈ 1.0039 × 2^(-15)

例子3：大数 65000
  FP32 指数位 E = 10001111₂ = 142₁₀
  实际指数 = 142 - 127 = 15
  意思是：65000 = 1.9849 × 2^15
```

**不同精度的指数位对比：**

| 精度类型 | 指数位数 | 偏移量 | 实际指数范围 | 数值范围 |
|---------|---------|--------|------------|---------|
| FP32 | 8位 | 127 | -126 到 +127 | ±3.4×10³⁸ |
| FP16 | 5位 | 15 | -14 到 +15 | ±65504 |
| BF16 | 8位 | 127 | -126 到 +127 | ±3.4×10³⁸ |

**关键观察：**
- **BF16 和 FP32 有相同的指数位（8位）**，所以数值范围相同
- **FP16 只有 5 位指数**，所以容易溢出（最大值只有 65504）

**指数位的作用总结：**

1. **决定数值范围**：指数位越多，能表示的数越大/越小
2. **不影响精度**：精度由尾数位决定
3. **防止溢出/下溢**：
   - 指数太小 → 数值范围小 → 容易溢出（FP16 的问题）
   - 指数够大 → 数值范围大 → 不易溢出（BF16 的优势）

**这就是为什么 BF16 比 FP16 更适合训练：**

虽然 BF16 的尾数位更少（精度略低），但指数位和 FP32 一样多（数值范围大，不易溢出）。

### 1.4 符号位详解

**符号位是什么？**

符号位（Sign bit）是浮点数表示中用来决定**数值正负**的部分，只占 1 位。

**工作原理：**

```
符号位 = 0 → 正数
符号位 = 1 → 负数

例如：
0 | 10000000 | 10010010... → +3.14159 (正数)
1 | 10000000 | 10010010... → -3.14159 (负数)
```

**实际例子：**

```
正数 π = 3.14159
  FP32: 0 | 10000000 | 10010010000111111011011
        ↑
      符号位=0 (正数)

负数 -π = -3.14159
  FP32: 1 | 10000000 | 10010010000111111011011
        ↑
      符号位=1 (负数)
```

**特殊情况：**

- **+0 和 -0**：浮点数有两个零
  - +0: 0 | 00000000 | 00000000000000000000000
  - -0: 1 | 00000000 | 00000000000000000000000
  - 在大多数运算中，+0 和 -0 被视为相等

- **+∞ 和 -∞**：无穷大也有正负
  - +∞: 0 | 11111111 | 00000000000000000000000
  - -∞: 1 | 11111111 | 00000000000000000000000

### 1.5 尾数位详解

**尾数位是什么？**

尾数位（Mantissa/Fraction）是浮点数表示中用来决定**数值精度**的部分，它存储数字的有效数字。

**核心概念：**

尾数位决定了数字能表示多少位有效数字，类似于科学计数法中的系数部分。

```
科学计数法（十进制）：
3.14159 = 3.14159 × 10^0
          ↑↑↑↑↑↑↑
          有效数字（7位）

浮点数（二进制）：
3.14159 ≈ 1.5708 × 2^1
          ↑↑↑↑↑
          尾数（由尾数位决定）
```

**隐含前导 1（Implicit Leading 1）：**

为了节省空间，浮点数使用了一个巧妙的技巧：

```
正规数的二进制科学计数法总是：1.xxxxx × 2^n

因为第一位总是 1，所以不需要存储！
只需要存储小数点后的部分（xxxxx）

例如：
实际值: 1.10010010000111111011011
存储值:  .10010010000111111011011 (省略前导1)
        ↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑
        这23位就是尾数位
```

**尾数位的工作原理：**

以 FP32 为例（23位尾数）：

```
尾数位: 10010010000111111011011

计算过程:
隐含前导1: 1.10010010000111111011011

展开二进制小数:
= 1 + 1×2⁻¹ + 0×2⁻² + 0×2⁻³ + 1×2⁻⁴ + 0×2⁻⁵ + 0×2⁻⁶ + 1×2⁻⁷ + ...
= 1 + 0.5 + 0 + 0 + 0.0625 + 0 + 0 + 0.0078125 + ...
= 1.5707963...

这个值就是尾数（Mantissa）
```

**不同精度的尾数位对比：**

| 精度类型 | 尾数位数 | 有效十进制位数 | 精度示例 |
|---------|---------|--------------|---------|
| FP32 | 23位 | ~7位 | 3.141592**7** |
| FP16 | 10位 | ~3位 | 3.14**1** |
| BF16 | 7位 | ~2位 | 3.1**4** |

**实际例子：π 的不同精度表示**

```
原始值: π = 3.14159265358979...

FP32 (23位尾数):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**尾数位的作用总结：**

1. **决定精度**：尾数位越多，能表示的有效数字越多
2. **不影响范围**：数值范围由指数位决定
3. **精度权衡**：
   - 尾数位多 → 精度高 → 占用空间大（FP32, FP16）
   - 尾数位少 → 精度低 → 占用空间小（BF16）

**为什么 FP16 比 BF16 精度高？**

```
FP16: 10位尾数 → ~3位十进制精度
BF16: 7位尾数  → ~2位十进制精度

但 BF16 的指数位更多（8位 vs 5位），所以数值范围更大！

这就是权衡：
- FP16: 高精度，小范围（容易溢出）
- BF16: 低精度，大范围（不易溢出）
```

**次正规数（Subnormal Numbers）：**

当指数位全为 0 时，尾数位的解释方式不同：

```
正规数（指数 ≠ 0）:
  隐含前导1: 1.xxxxx × 2^n

次正规数（指数 = 0）:
  没有隐含前导1: 0.xxxxx × 2^(最小指数)

作用：填补 0 附近的空隙，避免下溢

例如 FP16:
  最小正规数: 1.0 × 2^(-14) = 6.10×10⁻⁵
  最小次正规数: 0.0000000001 × 2^(-14) = 5.96×10⁻⁸
```

### 1.6 三者关系总结

**符号位、指数位、尾数位的协同工作：**

```
浮点数 = (-1)^符号位 × 2^(指数位-偏移量) × (1.尾数位)
         ↑            ↑                  ↑
      正负号        数值范围            精度

例如：π = 3.14159265

符号位 = 0 → 正数
指数位 = 128 → 实际指数 = 128-127 = 1 → 乘以 2^1
尾数位 = 10010010... → 1.5707963... → 有效数字

最终：(+1) × 2^1 × 1.5707963 = 3.14159274
```

**三者的分工：**

| 组成部分 | 作用 | 影响 | 类比 |
|---------|------|------|------|
| 符号位 | 决定正负 | 数值符号 | 正负号 |
| 指数位 | 决定范围 | 能表示多大/多小的数 | 科学计数法的指数 |
| 尾数位 | 决定精度 | 能表示多少位有效数字 | 科学计数法的系数 |

**设计权衡：**

```
总位数固定（如16位），如何分配？

方案1 - FP16: [1位符号] [5位指数] [10位尾数]
  优点: 精度高（10位尾数）
  缺点: 范围小（5位指数，最大65504）
  适合: 推理（数值范围可控）

方案2 - BF16: [1位符号] [8位指数] [7位尾数]
  优点: 范围大（8位指数，最大10^38）
  缺点: 精度低（7位尾数）
  适合: 训练（需要大范围，避免溢出）
```

**完整示例：-0.000123 的表示**

```
原始值: -0.000123

二进制: 1 | 01110000 | 00000010000011000100101
        ↑      ↑              ↑
      符号   指数(8位)      尾数(23位)

   实际指数 = 112 - 127 = -15
```

### 1.7 偏移量（Bias）详解

**偏移量是什么？**

偏移量（Bias）是指数位编码中使用的一个固定常数，用于将无符号整数转换为有符号的实际指数。

**为什么需要偏移量？**

指数需要表示正数和负数（如 2^15 和 2^(-15)），但指数位本身只能存储无符号整数（0-255）。偏移量解决了这个问题：

```
存储的指数值（无符号）→ 减去偏移量 → 实际指数（有符号）

例如 FP32:
  存储值 E = 0-255（无符号整数）
  偏移量 = 127
  实际指数 = E - 127（范围：-127 到 +128）
```

**不同精度的偏移量：**

| 精度类型 | 指数位数 | 偏移量计算 | 偏移量值 | 实际指数范围 |
|---------|---------|-----------|---------|------------|
| FP32 | 8位 | 2^(8-1) - 1 | 127 | -126 到 +127 |
| FP16 | 5位 | 2^(5-1) - 1 | 15 | -14 到 +15 |
| BF16 | 8位 | 2^(8-1) - 1 | 127 | -126 到 +127 |
| FP8 (E4M3) | 4位 | 2^(4-1) - 1 | 7 | -6 到 +7 |
| FP8 (E5M2) | 5位 | 2^(5-1) - 1 | 15 | -14 到 +15 |

**偏移量的计算公式：**

```
偏移量 = 2^(指数位数-1) - 1

例如：
FP32: 2^(8-1) - 1 = 2^7 - 1 = 128 - 1 = 127
FP16: 2^(5-1) - 1 = 2^4 - 1 = 16 - 1 = 15
```

**为什么是 2^(n-1) - 1？**

这个公式让指数范围尽可能对称：

```
FP32 (8位指数):
  存储范围: 0-255
  偏移量: 127
  实际范围: -127 到 +128

  对称分布:
    负指数: -127 到 -1 (127个值)
    零指数: 0 (1个值)
    正指数: +1 到 +128 (128个值)
```

**偏移量的实际应用：**

```
例子1：存储 2^5 = 32

1. 实际指数 = 5
2. 存储指数 = 5 + 127 = 132
3. 二进制: 10000100₂

解码时:
1. 读取存储指数 = 132
2. 实际指数 = 132 - 127 = 5
3. 计算: 1.xxx × 2^5 = 32.xxx

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

例子2：存储 2^(-10) = 0.0009765625

1. 实际指数 = -10
2. 存储指数 = -10 + 127 = 117
3. 二进制: 01110101₂

解码时:
1. 读取存储指数 = 117
2. 实际指数 = 117 - 127 = -10
3. 计算: 1.xxx × 2^(-10) = 0.000xxx
```

**特殊值的编码：**

偏移量系统中有两个特殊的存储值：

```
存储指数 = 0（全0）:
  → 表示次正规数或零
  → 实际指数固定为 (1 - 偏移量)
  → FP32: 实际指数 = 1 - 127 = -126

存储指数 = 最大值（全1）:
  → 表示无穷大或 NaN
  → FP32: 存储值 = 255
  → 不使用偏移量计算
```

**偏移量对比：FP16 vs BF16**

```
FP16 (偏移量=15):
  存储值 0   → 实际指数 = 0 - 15 = -15 (特殊：次正规数)
  存储值 1   → 实际指数 = 1 - 15 = -14
  存储值 15  → 实际指数 = 15 - 15 = 0
  存储值 30  → 实际指数 = 30 - 15 = 15
  存储值 31  → 实际指数 = 31 - 15 = 16 (特殊：无穷/NaN)

BF16 (偏移量=127):
  存储值 0   → 实际指数 = 0 - 127 = -127 (特殊：次正规数)
  存储值 1   → 实际指数 = 1 - 127 = -126
  存储值 127 → 实际指数 = 127 - 127 = 0
  存储值 254 → 实际指数 = 254 - 127 = 127
  存储值 255 → 实际指数 = 255 - 127 = 128 (特殊：无穷/NaN)

关键差异：
- BF16 的偏移量更大 → 可以表示更大/更小的指数
- 这就是为什么 BF16 数值范围和 FP32 一样大！
```

### 1.8 有效位数详解

**有效位数是什么？**

有效位数（Significant Digits）是指一个数字中能够准确表示的数字位数，决定了数值的精度。

**核心概念：**

```
十进制有效位数：
3.14159265 有 9 位有效数字
0.000123   有 3 位有效数字（前导零不算）
123000     有 3-6 位有效数字（取决于尾部零是否有意义）

二进制有效位数：
由尾数位数决定
```

**尾数位数 → 有效位数的转换：**

```
公式：十进制有效位数 ≈ 尾数位数 × log₁₀(2)
                      ≈ 尾数位数 × 0.301

例如：
FP32: 23位尾数 × 0.301 ≈ 6.9 ≈ 7位十进制有效数字
FP16: 10位尾数 × 0.301 ≈ 3.0 ≈ 3位十进制有效数字
BF16: 7位尾数  × 0.301 ≈ 2.1 ≈ 2位十进制有效数字
```

**不同精度的有效位数对比：**

| 精度类型 | 尾数位数 | 二进制有效位 | 十进制有效位 | 示例 |
|---------|---------|------------|------------|------|
| FP64 | 52位 | 53位 | ~16位 | 3.141592653589793 |
| FP32 | 23位 | 24位 | ~7位 | 3.141593 |
| FP16 | 10位 | 11位 | ~3位 | 3.14 |

**有效位数与机器精度（Machine Epsilon）：**

```
机器精度 = 2^(-尾数位数)

FP32: ε = 2^(-23) ≈ 1.19×10⁻⁷
  → 相邻两个数的最小间隔（在 1.0 附近）

FP16: ε = 2^(-10) ≈ 9.77×10⁻⁴
  → 精度比 FP32 低约 10000 倍

BF16: ε = 2^(-7) ≈ 7.81×10⁻³
  → 精度比 FP32 低约 65000 倍

实际意义:
  1.0 + ε = 1.0000001... (FP32 能区分)
  1.0 + ε/2 = 1.0 (FP32 无法区分，舍入为 1.0)
```

**有效位数的权衡：**

```
更多有效位数的优点:
  ✓ 精度高，误差小
  ✓ 适合科学计算
  ✓ 累积误差小

更多有效位数的缺点:
  ✗ 占用空间大
  ✗ 计算速度慢
  ✗ 带宽需求高

深度学习的选择:
  训练: 需要较高精度（FP32/BF16）避免梯度消失
  推理: 可以降低精度（FP16/INT8）提高速度
  权重: 可以用低精度（INT4）节省内存
  激活: 需要中等精度（FP16）保证准确性
```

**总结：偏移量 vs 有效位数**

| 概念 | 作用 | 影响的部分 | 决定因素 |
|------|------|-----------|---------|
| 偏移量 | 让指数能表示正负数 | 指数位 | 指数位数（2^(n-1)-1） |
| 有效位数 | 决定数值精度 | 尾数位 | 尾数位数（×0.301） |

```
完整公式:
浮点数 = (-1)^符号位 × 2^(指数位-偏移量) × (1.尾数位)
                              ↑              ↑
                          偏移量转换      有效位数决定精度
```

## 2. 不同精度的特点对比

### 2.1 FP32 (单精度浮点数)
**优点：**
- 精度高，数值稳定
- 训练收敛性好
- 传统深度学习的标准精度

**缺点：**
- 内存占用大（4 bytes/参数）
- 计算速度相对慢
- 带宽需求高

**适用场景：**
- 模型训练（尤其是小模型）
- 需要高精度的科学计算
- 调试和验证

### 2.2 FP16 (半精度浮点数)
**优点：**
- 内存占用减半（2 bytes/参数）
- GPU 计算速度快（Tensor Core 加速）
- 带宽需求减半

**缺点：**
- 数值范围小，容易溢出（上溢/下溢）
- 梯度消失问题严重
- 需要 Loss Scaling 技术

**适用场景：**
- 混合精度训练（FP16 + FP32）
- 推理加速
- 现代 GPU（V100+）

### 2.3 BF16 (Brain Float16)
**优点：**
- 保持 FP32 的数值范围（8位指数）
- 内存占用减半
- 不易溢出，训练更稳定
- 无需 Loss Scaling

**缺点：**
- 精度略低于 FP16（7位尾数 vs 10位）
- 需要硬件支持（A100+, TPU）

**适用场景：**
- 大模型训练（LLaMA, GPT 等）
- 推理加速
- Google TPU 和 NVIDIA A100+ GPU

### 2.4 FP8 (8位浮点数)
**优点：**
- 内存占用极小（1 byte/参数）
- 计算速度极快
- 适合超大模型

**缺点：**
- 精度损失明显
- 需要精细的量化策略
- 硬件支持有限（H100+）

**适用场景：**
- 推理加速（尤其是 Transformer）
- 超大模型部署
- NVIDIA H100 GPU

### 2.5 INT8 量化
**优点：**
- 内存占用最小（1 byte/参数）
- 推理速度快
- 广泛的硬件支持

**缺点：**
- 需要校准数据集
- 精度损失较大
- 训练困难（QAT）

**适用场景：**
- 边缘设备部署
- CPU 推理
- 移动端应用

### 2.6 INT4/INT2 极低精度
**优点：**
- 极致压缩（0.5 byte/参数）
- 适合资源受限环境

**缺点：**
- 精度损失严重
- 需要复杂的量化技术（GPTQ, AWQ）

**适用场景：**
- 超大模型压缩（70B+ 模型）
- 消费级硬件部署

## 5. 混合精度技术

### 5.1 训练中的混合精度

```python
# PyTorch AMP (Automatic Mixed Precision)
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for data, target in dataloader:
    optimizer.zero_grad()

    # 前向传播使用 FP16
    with autocast():
        output = model(data)
        loss = criterion(output, target)

    # 反向传播使用 FP32（梯度累积）
    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()
```

**原理：**
- 前向传播：FP16（快速）
- 权重存储：FP32（精确）
- 梯度计算：FP16 → FP32（稳定）
- 权重更新：FP32（避免下溢）

## 8. 量化技术深入

### 8.1 PTQ (Post-Training Quantization)

**原理：**
训练后量化，无需重新训练

**方法：**
```python
# 对称量化
Q = round(FP32 / scale)
scale = max(abs(FP32)) / 127

# 非对称量化
Q = round((FP32 - zero_point) / scale)
scale = (max(FP32) - min(FP32)) / 255
zero_point = -min(FP32) / scale
```

**代表技术：**
- GPTQ: 逐层量化，考虑 Hessian 矩阵
- AWQ: 激活感知量化，保护重要权重
- SmoothQuant: 平滑激活分布

### 8.2 QAT (Quantization-Aware Training)

**原理：**
训练时模拟量化，学习量化友好的权重

**优点：**
- 精度损失更小
- 适合极低精度（INT4）

**缺点：**
- 需要重新训练
- 计算成本高

### 8.3 vLLM 中的量化实现

```python
# vllm/model_executor/layers/quantization/awq.py
class AWQLinearMethod:
    def apply_weights(self, weights, x):
        # 1. 反量化
        weights_fp16 = weights.dequantize()

        # 2. 矩阵乘法
        output = torch.matmul(x, weights_fp16)

        return output
```

## 9. 精度相关的常见问题

### 9.1 数值溢出

**问题：**
FP16 范围小（±65504），容易溢出

**解决方案：**
```python
# Loss Scaling
loss = loss * scale_factor  # 放大 loss
loss.backward()
optimizer.step()
optimizer.zero_grad()
# 梯度会自动缩小
```

### 9.2 梯度消失

**问题：**
FP16 最小正数 ≈ 6×10⁻⁵，小梯度会变成 0

**解决方案：**
- 使用 BF16（范围更大）
- 使用混合精度（梯度用 FP32）
- 梯度裁剪

### 9.3 精度不匹配

**问题：**
不同层使用不同精度导致类型错误

**解决方案：**
```python
# vLLM 中的类型转换
def forward(self, x):
    # 确保输入类型匹配
    x = x.to(self.weight.dtype)
    output = F.linear(x, self.weight)
    return output
```

```
  FP16     | 65000.0  | 0         | 0         | 刚好在范围内
```

### 11.2 神经网络权重的精度转换

#### 实际权重矩阵例子

```python

weights_fp32 = torch.tensor([
    [ 0.0234567,  -0.1234567,   0.5678901,  -0.0001234],
    [-0.8765432,   0.0009876,  -0.3456789,   0.9876543],
    [ 0.0000123,  -0.0567890,   0.1111111,  -0.2222222],
    [ 0.7654321,  -0.4321098,   0.0098765,  -0.6543210]
], dtype=torch.float32)

# 转换为 FP16
weights_fp16 = weights_fp32.half()
# 结果:
# [[ 0.02344,  -0.1235,   0.5679,  -0.0001234],
#  [-0.8765,    0.0009876, -0.3457,   0.9878],
#  [ 0.00001228, -0.05679,  0.1111,  -0.2222],

weights_bf16 = weights_fp32.bfloat16()
# 结果:
# [[ 0.0234,   -0.1230,   0.5664,  -0.0001221],
#  [-0.8750,    0.0009918, -0.3438,   0.9844],
#  [ 0.00001216, -0.0566,   0.1113,  -0.2227],
#  [ 0.7656,   -0.4297,   0.009857, -0.6562]]

# INT8 量化
scale = weights_fp32.abs().max() / 127
# scale = 0.9876543 / 127 = 0.007776
weights_int8 = torch.round(weights_fp32 / scale).clamp(-128, 127).to(torch.int8)
# 结果:
# [[   3,  -16,   73,    0],
#  [-113,    0,  -44,  127],
#  [   0,   -7,   14,  -29],

weights_dequant = weights_int8.float() * scale
# 结果:
# [[ 0.0233,  -0.1244,   0.5677,   0.0000],
#  [-0.8787,   0.0000,  -0.3422,   0.9876],
#  [ 0.0000,  -0.0544,   0.1089,  -0.2255],
#  [ 0.7621,  -0.4355,   0.0078,  -0.6532]]

# 误差分析

# error_bf16 = 0.000089 (略大于 FP16)

error_int8 = (weights_fp32 - weights_dequant).abs().mean()
```

### 11.5 KV Cache 的精度影响

#### Attention 计算例子

```python
# Query, Key, Value (seq_len=4, hidden_dim=8)
Q = torch.randn(4, 8, dtype=torch.float32)
K = torch.randn(4, 8, dtype=torch.float32)
V = torch.randn(4, 8, dtype=torch.float32)

# FP32 Attention
scores_fp32 = torch.matmul(Q, K.T) / math.sqrt(8)
attn_fp32 = F.softmax(scores_fp32, dim=-1)
output_fp32 = torch.matmul(attn_fp32, V)

# FP16 KV Cache
K_fp16 = K.half()
V_fp16 = V.half()
scores_fp16 = torch.matmul(Q.half(), K_fp16.T) / math.sqrt(8)
attn_fp16 = F.softmax(scores_fp16, dim=-1)
output_fp16 = torch.matmul(attn_fp16, V_fp16)

# 误差分析
error = (output_fp32 - output_fp16.float()).abs().mean()
# error ≈ 0.0001 ~ 0.001 (可接受)

# 内存节省
memory_fp32 = K.numel() * 4 + V.numel() * 4  # 256 bytes
memory_fp16 = K_fp16.numel() * 2 + V_fp16.numel() * 2  # 128 bytes
# 节省 50% 内存！

# FP8 KV Cache (实验性)
# 进一步节省到 64 bytes，但精度损失更大
```

## 12. 总结

### 核心要点：
1. **精度是内存、速度、准确性的权衡**
2. **BF16 是当前大模型的最佳选择**（训练+推理）
3. **INT8/INT4 适合资源受限场景**（边缘部署）
4. **混合精度是最优策略**（不同组件用不同精度）
5. **硬件支持是关键**（选择精度要考虑 GPU 能力）

### 实践建议：
- 训练：BF16 混合精度（A100+）
- 推理：BF16（云端）或 INT8（边缘）
- 超大模型：INT4 量化（GPTQ/AWQ）
- 调试：FP32（精度最高）
