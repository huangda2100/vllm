# 推理加速硬件选择指南

## 1. 硬件选择的核心考量

### 1.1 关键指标

在选择推理硬件时，需要综合考虑以下指标：

| 指标 | 说明 | 重要性 |
|------|------|--------|
| **算力（TFLOPS/TOPS）** | 每秒浮点/整数运算次数 | ⭐⭐⭐⭐⭐ |
| **内存容量（GB）** | 可加载的模型大小 | ⭐⭐⭐⭐⭐ |
| **内存带宽（TB/s）** | 数据传输速度 | ⭐⭐⭐⭐⭐ |
| **精度支持** | FP32/FP16/BF16/INT8/INT4 | ⭐⭐⭐⭐ |
| **功耗（W）** | 能耗成本 | ⭐⭐⭐⭐ |
| **价格（$）** | 采购成本 | ⭐⭐⭐⭐ |
| **互连带宽** | 多卡通信速度 | ⭐⭐⭐ |

### 1.2 性能瓶颈分析

```
推理性能的三大瓶颈：

1. 计算瓶颈（Compute-bound）
   - 特征：GPU 利用率高（>80%）
   - 表现：大量矩阵运算
   - 优化：提升算力（TFLOPS）
   - 适合：大批量推理

2. 内存瓶颈（Memory-bound）
   - 特征：内存带宽打满
   - 表现：频繁读写权重
   - 优化：提升带宽（TB/s）
   - 适合：小批量推理、大模型

3. 通信瓶颈（Communication-bound）
   - 特征：多卡间通信慢
   - 表现：张量并行、流水线并行
   - 优化：提升互连带宽（NVLink）
   - 适合：超大模型多卡推理
```

### 2.2 Ampere 架构（A 系列）

#### A100 (2020)

**核心规格：**
```
GPU 架构：Ampere
制程工艺：7nm (TSMC)
晶体管数：542 亿
CUDA 核心：6912
Tensor Core：432 (第3代)

算力：
- FP32: 19.5 TFLOPS
- TF32: 156 TFLOPS (Tensor Core)
- FP16: 312 TFLOPS (Tensor Core)
- BF16: 312 TFLOPS (Tensor Core)
- INT8: 624 TOPS (Tensor Core)
- INT4: 1248 TOPS (Tensor Core)

内存：
- 容量：40GB / 80GB HBM2e
- 带宽：1.6 TB/s (40GB) / 2.0 TB/s (80GB)
- 位宽：5120-bit

互连：
- NVLink 3.0: 600 GB/s (12 links)
- PCIe 4.0: 64 GB/s

功耗：
- TDP: 400W (SXM) / 250W (PCIe)

价格：
- ~$10,000 - $15,000
```

**特性：**
```
✓ 首次支持 TF32（无需修改代码，自动加速）
✓ 第3代 Tensor Core（支持 FP64, TF32, BF16, FP16, INT8, INT4）
✓ 多实例 GPU（MIG）：可分割为 7 个独立实例
✓ 结构化稀疏（2:4 稀疏，2× 加速）
✓ 成熟稳定，生态完善

适用场景：
- 通用训练和推理
- 中小型模型推理（<70B）
- 成本敏感的生产环境
- 多租户场景（MIG）
```

**推理性能（LLaMA-7B）：**
```
FP16 推理：
- 吞吐量：~150 tokens/s
- 延迟：~6.7 ms/token
- 批量大小：128
- 内存占用：14 GB

INT8 推理：
- 吞吐量：~200 tokens/s
- 延迟：~5 ms/token
- 批量大小：256
- 内存占用：7 GB
```

### 2.3 Hopper 架构（H 系列）

#### H100 (2022)

**核心规格：**
```
GPU 架构：Hopper
制程工艺：4nm (TSMC)
晶体管数：800 亿
CUDA 核心：16896
Tensor Core：528 (第4代)

算力：
- FP32: 67 TFLOPS
- TF32: 989 TFLOPS (Tensor Core)
- FP16: 1979 TFLOPS (Tensor Core)
- BF16: 1979 TFLOPS (Tensor Core)
- FP8: 3958 TFLOPS (Tensor Core) ⭐ 新增
- INT8: 3958 TOPS (Tensor Core)

内存：
- 容量：80GB HBM3
- 带宽：3.35 TB/s (相比 A100 提升 67%)
- 位宽：5120-bit

互连：
- NVLink 4.0: 900 GB/s (18 links, 提升 50%)
- PCIe 5.0: 128 GB/s (提升 100%)

功耗：
- TDP: 700W (SXM) / 350W (PCIe)

价格：
- ~$30,000 - $40,000
```

**核心特性：**
```
⭐ FP8 精度支持（Transformer Engine）
   - 相比 FP16 加速 2×
   - 精度损失 <1%
   - 自动混合精度（FP8 + FP16）

⭐ 第4代 Tensor Core
   - 支持 FP64, TF32, BF16, FP16, FP8, INT8
   - FP8 专用加速单元
   - 稀疏加速（2:4 稀疏）

⭐ Transformer Engine
   - 自动 FP8 量化
   - 动态范围调整
   - 针对 Transformer 优化

⭐ DPX 指令（动态编程加速）
   - 加速动态规划算法
   - 用于序列对齐、路径搜索

⭐ 线程块集群（Thread Block Clusters）
   - 更好的 SM 协作
   - 提升大规模并行效率

⭐ 异步执行（Asynchronous Execution）
   - Tensor Memory Accelerator (TMA)
   - 减少 CPU 开销
```

**推理性能提升：**
```
相比 A100：

LLaMA-7B FP16：
- A100: 150 tokens/s
- H100: 450 tokens/s
- 加速：3× ⭐

LLaMA-7B FP8：
- H100: 900 tokens/s
- 加速：6× vs A100 ⭐⭐

LLaMA-70B FP16（4卡张量并行）：
- A100: 80 tokens/s
- H100: 240 tokens/s
- 加速：3×

内存带宽提升：
- A100: 2.0 TB/s
- H100: 3.35 TB/s
- 提升：67%（对小批量推理关键）
```

**适用场景：**
```
✓ 大模型推理（70B+）
✓ 高吞吐量服务（FP8 加速）
✓ 低延迟推理（高带宽）
✓ 多模态模型
✓ 前沿研究和开发
```

## 3. 硬件选择决策树

### 3.1 按模型规模选择

```
模型规模决策树：

小模型（<7B）
├─ 预算充足 → H100/H200（FP8 极速）
├─ 预算中等 → A100（FP16 够用）
└─ 预算紧张 → T4/L4（INT8 推理）

中型模型（7B-70B）
├─ 单卡运行
│   ├─ FP16 → H200（141GB）
│   ├─ INT8 → H100（80GB）
│   └─ INT4 → A100（80GB）
└─ 多卡运行
    ├─ 2-4卡 → H100（高带宽）
    └─ 4-8卡 → A100（成本优）

大模型（70B-405B）
├─ 单节点
│   ├─ FP8 → GB200（576GB）
│   ├─ INT4 → H200 4卡
│   └─ INT4 → H100 8卡
└─ 多节点
    └─ H100/H200 集群

超大模型（405B+）
└─ GB200 集群（唯一选择）
```

### 3.2 按场景选择

```
在线服务（延迟敏感）：
推荐：H100 > H200 > A100
原因：
- 高带宽（小批量推理快）
- FP8 支持（H100）
- 低延迟

离线批处理（吞吐量优先）：
推荐：A100 > H100
原因：
- 性价比高
- 大批量推理
- 成本敏感

研究开发：
推荐：H100 > H200 > GB200
原因：
- 最新特性（FP8）
- 大内存（长上下文）
- 灵活性高

生产部署（成本敏感）：
推荐：A100 > L4 > T4
原因：
- 成熟稳定
- 生态完善
- TCO 低
```
